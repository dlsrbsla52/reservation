# 알림 (Notification) 기능 목록

현재 프로젝트에 알림 시스템이 부재하며, 계약 갱신·만료·결제 등 비즈니스 이벤트에 대한 알림 인프라가 필요합니다.

## 추가 기능 목록

### 1. 이메일 알림 (우선순위: 높음)

| 기능 | 설명 |
|------|------|
| 이메일 인증 메일 | 회원가입 시 인증 링크 발송 (IAM 모듈 연동) |
| 계약 생성 확인 메일 | 계약 생성 시 계약 요약 정보 발송 |
| 계약 만료 사전 안내 | 만료 30일/7일/1일 전 갱신 안내 메일 (`renewal_notified_at` 기록) |
| 계약 갱신 완료 메일 | 자동/수동 갱신 완료 후 신규 계약 정보 발송 |
| 결제 안내 메일 | 결제 청구 생성 시 납부 안내 |
| 연체 안내 메일 | 납부 기한 초과 시 연체 알림 |
| 예약 접수 확인 메일 | 예약 생성 시 접수 확인 |
| 상담 완료 안내 메일 | 상담 처리 완료 후 결과 안내 |

### 2. 알림 모듈 설계 (우선순위: 중간)

| 기능 | 설명 |
|------|------|
| `notification` 모듈 신설 | 알림 발송 책임 분리. 이메일·SMS·푸시를 단일 인터페이스로 추상화 |
| 알림 이력 저장 | 발송 결과(성공/실패), 발송 시각, 수신자, 템플릿 종류 기록 |
| 알림 재발송 | 발송 실패 건 자동 재시도 (Exponential Backoff) |
| 템플릿 관리 | HTML 이메일 템플릿을 DB 또는 파일로 관리, 다국어 지원 고려 |

### 3. SMS 알림 (우선순위: 중간)

| 기능 | 설명 |
|------|------|
| SMS 계약 만료 안내 | 이메일 미열람 회원 대상 SMS 폴백 |
| SMS 결제 안내 | 결제 청구·연체 SMS 발송 |
| SMS 게이트웨이 연동 | 네이버 SENS 또는 AWS SNS 기반 |

### 4. 인앱/웹 푸시 알림 (우선순위: 낮음)

| 기능 | 설명 |
|------|------|
| 알림 센터 API | `GET /api/v1/notification` — 읽지 않은 알림 목록 조회 |
| 알림 읽음 처리 | `PUT /api/v1/notification/{id}/read` |
| 전체 읽음 처리 | `PUT /api/v1/notification/read-all` |
| WebSocket 실시간 알림 | 관리자 대시보드용 실시간 이벤트 수신 |
